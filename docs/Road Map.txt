Phase 0 — Make the Shell Talk to the Backend (1–2 weeks)
Goal: One real page that proves frontend ↔ API ↔ IIS proxy works.

Deliverables

Add src/lib/api-client.ts — thin fetch wrapper for /api/* (same origin via IIS).
Wire one page — e.g. /ai-decision-engine/recommendations or a new /decision-workbench:
GET /api/trading-universe/symbols
POST /api/decisioning/hybrid-evaluate
Replace hardcoded metrics on that page with API response fields (Recommendation, ConfidenceScore, NoTradeReasons, Evidence).
Add loading / error states (you already have placeholders in command-center-page.tsx).
Why first: Everything else depends on this. No DB, no MT5, no auth required.

Phase 2 — Platform Foundation (3–4 weeks)
Goal: Shared infrastructure so features stop being one-off.

Deliverables

Database choice — SQL Server 2022 (already in product copy) or PostgreSQL; EF Core in Cacsms.Engine.Infrastructure/Persistence.
Core entities — DecisionRecord, WorkflowStageStatus, AuditEvent, TradingAccount (minimal).
ASP.NET Identity + RBAC — align with roles in navigation.ts (Super Administrator, Trader, Risk Manager, etc.).
API conventions — validation, problem details, correlation IDs (Serilog already wired).
Frontend data layer — TanStack Query (declared in stack doc, not in package.json yet) + optional Zustand for filters.
CORS / env config — NEXT_PUBLIC_API_BASE for dev when not behind IIS.
Success criteria

Login required for API (except /health).
Role gates which nav modules appear (replace hardcoded activeRole = "Super Administrator").
Decisions and audit events survive restarts.

Phase 6 — Risk, Reporting & Learning (ongoing)
Goal: Match the governance story in the nav catalog.

Prioritized slices
